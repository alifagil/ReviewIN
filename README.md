# 🚀 ReviewIN – Fake Review Detection System

ReviewIN is a web-based Fake Review Detection System developed to help users identify potentially fraudulent product reviews on e-commerce platforms such as Amazon.

The system combines **Natural Language Processing (NLP)**, **Machine Learning**, **Web Scraping**, and **Explainable AI** to analyze review authenticity, classify reviews as **BOT** or **HUMAN**, generate a **Fraud Score**, and provide insights into suspicious review patterns.

# 📖 Project Overview

Online reviews significantly influence purchasing decisions in modern e-commerce platforms. However, the increasing presence of fake reviews generated by bots, click farms, and coordinated manipulation campaigns has made it difficult for consumers to determine review authenticity.

ReviewIN was developed as an automated solution capable of:

* Collecting review data
* Cleaning and processing review text
* Detecting suspicious reviews
* Generating Fraud Scores
* Presenting analysis results through an interactive dashboard

The project was developed as part of a Software Engineering academic project focusing on practical Machine Learning implementation.

# ✨ Core Features

## 🔗 Amazon URL Analysis

Users can submit an Amazon product URL.

The system will:

* Extract reviews using Selenium
* Build a review dataset
* Analyze review authenticity
* Generate Fraud Score results

---

## 📝 Manual Review Analysis

Users can paste review text directly into the system.

This mode is useful when:

* Scraping is unavailable
* Users already possess exported review data
* Bulk review analysis is required

---

## 🧹 Regex-Based Review Processing

ReviewIN includes a custom Regex Engine capable of:

### Review Segmentation

Separating large blocks of copied Amazon reviews into individual review entries.

### Metadata Stripping

Removing:

* Ratings
* Dates
* Locations
* Verified Purchase labels
* Product variations
* Other Amazon-specific metadata

This preprocessing stage improves consistency before Machine Learning inference.

---

## 🤖 Machine Learning Classification

The system classifies each review as:

* HUMAN
* BOT

using a trained Logistic Regression model.

---

## 📊 Fraud Score Generation

Fraud Score is calculated using:

```text
Fraud Score = (BOT Reviews / Total Reviews) × 100%
```

The score provides a quick indication of potential review manipulation.

---

## 🔍 Explainable AI

ReviewIN includes a lightweight Explainable AI module.

The system analyzes Logistic Regression feature weights and highlights suspicious keywords that contribute to BOT predictions.

This improves transparency and interpretability.

---

## 📈 Interactive Dashboard

The dashboard displays:

* Fraud Percentage
* Human Percentage
* BOT Percentage
* Doughnut Chart Visualization
* Review Classification Table
* Fraud Indicators

---

## TF-IDF Feature Engineering

TF-IDF converts review text into numerical vectors based on word importance.

This representation is then used as input for the classification model.

---

## Logistic Regression Classification

The model performs binary classification:

```text
0 = HUMAN
1 = BOT
```

The classifier was selected due to:

* Fast inference time
* Simplicity
* Interpretability
* Strong performance on text classification tasks

---

## Explainable AI Module

The Explainable AI component extracts influential keywords based on Logistic Regression coefficient weights.

These indicators help explain suspicious classifications.

## Scraper Limitations

The Selenium-based scraper is operational and capable of collecting Amazon reviews.

However, Amazon frequently updates:

* Anti-bot protection mechanisms
* CAPTCHA systems
* Dynamic page structures
* Request restrictions

As a result, scraping reliability may vary over time.

To mitigate these limitations, ReviewIN includes:

* Stealth browser configuration
* Human-like scrolling behavior
* Backup simulation mechanism for demonstration purposes

Future development will focus on improving scraping resilience and adaptability.
